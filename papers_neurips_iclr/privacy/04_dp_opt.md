## Abstract

Large language models are increasingly used via cloud APIs, but prompt tuning with sensitive data creates privacy risks. The authors propose DP-OPT (Differentially-Private Offsite Prompt Tuning), a framework where discrete prompts are optimized locally on the client side using smaller models, then transferred to cloud-based LLMs. The key innovation is the first private prompt generation mechanism, achieved through a differentially-private ensemble of in-context learning with private demonstrations. Experiments demonstrate that prompts generated by smaller local models (Vicuna-7B) transfer effectively to larger commercial systems (DaVinci-003) while maintaining differential privacy guarantees and achieving competitive results compared to non-private alternatives.

## 1. Introduction

### 1.1 The Privacy Problem in Prompt Tuning

Organizations want to leverage powerful cloud LLMs (GPT-3.5, GPT-4) but face a dilemma:
- **Prompt tuning requires data:** Demonstrations/exemplars must be sent to the API
- **Data may be sensitive:** Medical records, financial data, proprietary content
- **API providers see prompts:** No guarantee against data extraction or logging
- **Prompts can leak data:** Optimized prompts may contain verbatim training data

### 1.2 Key Insight

Discrete prompts optimized on small local models can transfer to larger cloud models. By making the optimization process differentially private, the transferred prompts provide formal privacy guarantees.

## 2. Method: DP-OPT

### 2.1 Architecture Overview

```
Client Side (Local, Private):
  - Small LLM (e.g., Vicuna-7B)
  - Private dataset
  - DP prompt optimization
  -> Outputs: Privacy-protected discrete prompt

Cloud Side (Remote, Untrusted):
  - Large LLM (e.g., DaVinci-003)
  - Receives only the private prompt
  - Generates predictions
```

### 2.2 Transferable Discrete Prompts

Using the Deep Language Network (DLN) method:
1. **Forward pass:** Local model predicts labels on training samples
2. **Backward pass:** Generate candidate prompts based on errors
3. **Selection:** Choose best-performing prompt

Key finding: "DLN-1 attains 8% accuracy gains after transfer to DaVinci-003 on average" -- prompts from small models work on large ones.

### 2.3 Privacy Mechanisms

**Private Prompt Generation:**
- Partition private data into K disjoint subsets
- Generate candidate prompts from each subset independently
- Use LimitedDomain mechanism to select tokens with privacy guarantees
- Each token selection satisfies epsilon_0-differential privacy

**Private Prompt Selection:**
- Apply the exponential mechanism to choose the best prompt from candidates
- Utility function: accuracy on a validation subset
- Selection satisfies differential privacy through the exponential mechanism

### 2.4 Privacy Analysis

**Privacy cost:** Grows as O(sqrt(m) * epsilon_0), where:
- m = number of generated tokens in the prompt
- epsilon_0 = per-token privacy budget

**Composition:** Uses Renyi Differential Privacy (RDP) for tighter composition bounds, convertible to standard (epsilon, delta)-DP.

## 3. Experimental Results

### 3.1 Classification Benchmarks

**Tasks:** SST-2 (sentiment), TREC (question type), MPQA (opinion), Disaster (relevance)

| Method | SST-2 | TREC | MPQA | Disaster | Privacy |
|---|---|---|---|---|---|
| Non-private ICL | 94.7% | 76.0% | 87.5% | 83.2% | None |
| DP-OPT (DaVinci-003) | 92.2% | 72.1% | 84.3% | 80.1% | epsilon=8 |
| Zero-shot | 80.0% | 65.0% | 78.0% | 72.0% | Perfect |

### 3.2 Transfer Learning

Prompts trained on Vicuna-7B transfer successfully across architectures:
- Vicuna-7B -> DaVinci-003: Consistent improvement over zero-shot
- Robustness across model families and sizes
- Performance gap with non-private baseline is modest

### 3.3 Privacy-Utility Tradeoff

Ablation across privacy budgets:
- **Tight budgets (small epsilon):** Accuracy degrades more on smaller target models
- **Moderate budgets:** Near-non-private performance on large models
- **Large models benefit:** Superior zero-shot capabilities provide a safety net

## 4. Analysis

### 4.1 Verbatim Data Leakage

Non-private prompt optimization can produce prompts containing verbatim copies of training data. Under DP noise:
- Verbatim copies are substantially reduced
- Remaining prompt tokens are randomized
- Formal guarantees prevent reconstruction attacks

### 4.2 Model IP Protection

The framework also protects model intellectual property:
- Cloud model parameters never reach the client
- Client's private data never reaches the cloud
- Only the privacy-protected prompt crosses the boundary

### 4.3 Interesting Observations

- Non-private OPT sometimes outperforms DLN-1 due to ensemble noise improving generalization
- Prompt diversity under DP can act as regularization
- Larger target models are more robust to noisy prompts

## 5. Threat Model

**Adversary capabilities:**
- Has access to the cloud model API
- Seeks to perform membership inference or data extraction
- May analyze the transferred prompt for private information

**Protection guarantees:**
- Transferred prompts satisfy (epsilon, delta)-differential privacy
- Any analysis of the prompt has bounded information gain about individual data points
- Composition ensures privacy across multiple prompt transfers

## 6. Related Work

- **Differential privacy:** DP-SGD (Abadi et al., 2016), private prediction (Dwork et al., 2006)
- **Prompt tuning:** Soft prompts (Lester et al., 2021), discrete optimization (Shin et al., 2020)
- **Private NLP:** DP text generation, private fine-tuning
- **Offsite tuning:** Training on local models for transfer to cloud models

DP-OPT uniquely combines discrete prompt optimization with differential privacy for cross-model transfer.

## 7. Conclusion

DP-OPT demonstrates that privacy-preserving prompt engineering is practical: prompts generated locally with differential privacy guarantees transfer effectively to larger cloud models. The approach achieves competitive accuracy while providing formal privacy protection, addressing a genuine practical need for organizations managing sensitive data while leveraging state-of-the-art LLMs.
