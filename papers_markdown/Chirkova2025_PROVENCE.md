# PROVENCE: Efficient and Robust Context Pruning for Retrieval-Augmented Generation

- **Authors:** Nadezhda Chirkova, Thibault Formal, Vassilina Nikoulina, Stephane Clinchant
- **Year:** 2025
- **Venue:** ICLR 2025
- **ArXiv:** [2501.16214](https://arxiv.org/abs/2501.16214)

## Abstract

PROVENCE addresses computational overhead in retrieval-augmented generation by removing irrelevant parts of retrieved contexts before LLM generation. Unlike existing methods that lack universality across diverse scenarios, PROVENCE dynamically determines the appropriate amount of pruning needed per instance and operates across multiple domains without domain-specific tuning. It formulates context pruning as sequence labeling, unifies pruning with context reranking in a single model, and leverages diverse training datasets for robustness. The method achieves context pruning with negligible to no performance drop across various domains and settings.

## Key Contributions

1. **Sequence labeling formulation**: Context pruning cast as binary token-level classification using DeBERTa, with "sentence rounding" at inference (selecting entire sentences if >50% of tokens score above threshold).
2. **Unified pruning + reranking**: Single forward pass with two task heads -- reranking (scalar score at BOS token) and pruning (per-token predictions), eliminating computational overhead of separate models.
3. **Diverse training for robustness**: Trained on MS MARCO (370K queries, 3.2M documents) with silver labels from Llama-3-8B-Instruct citation-based annotation (~90% success rate).
4. **Adaptive compression ratio**: Dynamically determines how much to prune per instance rather than using a fixed ratio.

## Methodology

- Fine-tunes DeBERTa to encode query-context pairs and output binary masks for irrelevant content
- Cross-encoding provides richer representations than independent sentence processing
- Training labels generated by prompting Llama-3-8B-Instruct to answer questions while citing relevant sentences
- Combined loss: binary cross-entropy for pruning + MSE distillation from reranker
- At inference, sentence rounding applies the token-level predictions at sentence granularity

## Key Results

| Benchmark | LLM-Eval Score | Compression |
|-----------|---------------|-------------|
| Natural Questions | 72.4% | 62% pruned |
| HotpotQA | 56.7% | 66% pruned |
| BioASQ | 80.6% | 49% pruned |

- Consistently matches or exceeds RECOMP and LLMLingua-2 at comparable compression ratios
- Achieves 2x generation speedups on large batch sizes
- Adaptive per-instance pruning ratio with single threshold parameter
- Operates with minimal computational cost in standard RAG pipelines

## Relevance to Thesis

PROVENCE's sequence labeling approach to context pruning is directly relevant to the thesis's compressor design, offering a lighter-weight alternative to LLMLingua-2's token classification. The unified pruning + reranking architecture parallels the thesis's P3 (joint) pipeline that combines retrieval and compression. PROVENCE's adaptive compression ratio per instance addresses the same concern as hypothesis H2 (coordination cliff) -- that a fixed compression ratio may be too aggressive for some contexts while insufficient for others. The training approach using citation-based silver labels from an instruction-tuned LLM could inform future improvements to the thesis's ICAE training procedure.
